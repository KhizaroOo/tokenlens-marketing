How data stays safe

Three checks before anyone sees data

CHECK 1 · EDGE

The gate

A JWT in an httpOnly session cookie is verified before any portal page loads.

→
CHECK 2 · SERVER

The page

The server re-checks the session on every protected page — defence in depth.

→
CHECK 3 · DATA

Your data only

Every query is scoped to your organization. Roles: viewer / admin / owner.

Provider API keys are encrypted with AES-256-GCM and decrypted only in memory at sync time — never stored in plain text.

Privacy by design

We store the numbers, not your content

  • Metadata only — never prompt text, AI responses, or code content
  • Encrypted provider credentials (AES-256-GCM)
  • Organization-scoped queries — each company sees only its own data
  • Built on enterprise security principles — certifications on the roadmap, not yet held
How it works

Two simple flows

Getting your AI numbers
1Add a provider's admin API key in Settings
2It's encrypted at rest (AES-256-GCM)
3A scheduled sync worker fetches usage from the provider
4Cost & usage appear on your dashboard (sync is scheduled, not real-time)
When someone contacts us
1They submit the contact or demo form
2Validation + rate-limit + honeypot block spam
3Stored safely — only a hashed IP, never the raw IP
4An email notification reaches the team (when email env is configured)
